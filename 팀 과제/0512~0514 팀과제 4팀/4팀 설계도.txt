설계서
파트분배
- 설계서, 베이스(변수명,메서드명), 프로그램 종료, 전체 정리 : 강두완
- 관리자 : 노기봉
- 학생생성 : 이광원, 한진만
- 학생목록출력, 학생1명출력 : 강두완, 곽대환, 홍성민
- 학생점수변경 : 김임형, 노기봉

class명 : Student
1. 멤버변수
- 인스턴스 변수 : 학생번호, 이름, 점수
- 클래스 변수(static) : 없음 // 모두가 공유할 공유자원이 없다.
2. 생성자
- input : 학생번호,이름, 점수
- output : 없음
- 기능 : 이름과 점수를 저장
3. 메서드
1) 학생1명출력
 - input : 번호
 - output : 없음
 - 기능 : 학생 정보 출력
2) 학생점수변경
 - input : 점수
 - output : 없음
 - 기능 : 점수 저장
3) 학생의 점수를 출력해주는 메서드

main
1. 스캐너
2. 멤버변수
- 목차선택 입력값
- 학생부 == Student배열[3]
- 저장된 학생 수 카운트용 변수
- 추가로 입력값이 있을 때 사용할 변수
3. main 기능
1) 목차생성 => 무한반복(사용자용)

	=== 학생부 프로그램 ===
	1. 학생목록출력
	2. 학생1명출력
	3. 프로그램 종료

 0 목차 선택 입력받기

 1 학생목록출력
 -> 기록된 학생정보가 없다면 정보가 없다고 출력
 -> 반복문을 통해 입력된 학생들의 기록 출력

 2 학생1명출력
 -> 기록된 학생정보가 없다면 정보가 없다고 출력
 -> 학생번호 받아서 해당 학생 정보 출력 메서드 실행
 ==> 학생번호 유효성 검사

 3 프로그램 종료
 -> 종료 메세지 출력

 4 관리자 모드 -> 하단에 별도 첨부
 -> 특정 값을 입력하면 관리자과정 출력

 5 잘못된 입력값 (유효성 검사)
 -> 잘못된 입력값이라고 출력

관리자 프로그램 부분
- 비밀번호 별도 체크
- 입력받은 비밀번호 유효성 체크
- 목차생성 => 종료할때까지 무한반복

	=== 관리자 프로그램 ===
	1. 학생생성
	2. 학생목록출력
	3. 학생점수변경
	4. 관리자 모드 종료

1 학생생성
-> 기록된 학생수를 통해서 생성 가능 여부 체크
==> 생성 불가능하다면 불가능하다고 출력
-> 이름 입력받기 (완료할때까지 무한반복)
==> 유효성 검사
-> 점수 입력받기 (완료할때까지 무한반복)
==> 유효성 검사
-> 학생정보 저장 메서드 실행
==> 메서드 실행하고 index++(저장된 학생수)

2 학생 목록 출력
-> 사용자용으로 만든거 그대로 복사해서 붙여넣음

3 학생점수출력
-> 입력된 학생이 없을경우
==> 학생목록이 없다고 출력
-> 몇번 학생을 선택할건지 질문
==> 입력된 번호 유효성 검사 == 무한반복문
-> 해당 학생의 점수 알려주고 변경할 점수 입력받기
==> 입력된 점수 유효성 검사 == 무한반복문
-> 점수저장하고 저장완료 메세지 출력

4 관리자모드 종료
-> 사용자 모드로 돌아감

5 잘못된 입력값(유효성 검사)
-> 잘못된 입력값이라고 출력

추가
모든 입력코드에 버퍼처리
초보자도 알 수 있을 만큼 주석
