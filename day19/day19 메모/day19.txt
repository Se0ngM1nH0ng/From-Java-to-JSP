mvc 패턴 

1. 설계서는 사실에 의거해서 정확하게 표기 한다.

2. 간단정리를 따로 해서 표기 한다.
(내가 아닌 타인이 볼 때 쉽게 알 수 있게 하도록)
(특히 고객사는 코드를 읽을 수 없기 때문에)

3. 자료형 (VO의 타입이)가 같다면,
DAO를 분리 하지 않는다.

4. 장바구니는 사용자 개개인이 따로 각각 봐야하니까
컨트롤러에서 구현하는게 맞다.
(상품 자체는 전체가 보는거라 DAO에서 구현하는게 맞지만..)

예)
오늘 하루 추천 상품 배너가 뜰 때
그걸 ctrl에서 구현하지않고
DAO에서 구현하면
어떤 한사람이 그걸 안보기 버튼을 누르면
모두가 보지않게 된다.

5. 메서드 시그니쳐가 변경되면, (모든 파트의 함수)
다른 파트에 영향을 줘버리게 된다.
-> 이것은 결합도가 높은 코드임을 의미 !
== 유지 보수 불리
== 안좋은 코드

=> 설계서를 꼼꼼히 짜도록 하기.

6. MVC 정리
1) 결합도를 낮추기 위해 CRUD 인자를 VO로 통일 !
2) CRUD output boolean으로 통일 !
3) 장바구니(카트), 로그인한 회원정보 등
-> Controller 에서 구현 하기.
4) VO가 여러개인 프로젝트.
